# Digital FTE Task Management Skills

Skills for managing tasks in the AI Employee system.

## Overview

The Digital FTE system uses an Obsidian vault with the following structure:

```
ai_employee_vault/
├── Inbox/           # Raw incoming items
├── Needs_Action/    # Items requiring action
├── Plans/           # Generated action plans
├── Pending_Approval/# Awaiting human approval
├── Approved/        # Approved actions
├── Rejected/        # Rejected actions
├── Done/            # Completed tasks
├── Logs/            # Activity logs
├── Accounting/      # Financial records
├── Briefings/       # CEO briefings
├── Dashboard.md     # Main dashboard
├── Company_Handbook.md  # Rules and guidelines
└── Business_Goals.md    # Business objectives
```

## Skills

### List Pending Tasks

List all tasks awaiting action in the Needs_Action folder.

```bash
# Using Claude Code with filesystem MCP
/list_files path="ai_employee_vault/Needs_Action"
```

### Read Task File

Read a specific task file to understand what needs to be done.

```bash
/read_file path="ai_employee_vault/Needs_Action/EMAIL_abc123_2026-01-07.md"
```

### Create Action Plan

Create a structured plan for completing a task.

```markdown
# Create Plan.md in Plans/ folder

---
type: plan
task_file: "TASK_FILENAME.md"
task_type: "email"
priority: "normal"
created: 2026-01-07T10:30:00Z
status: planned
---

# 📋 Action Plan: TASK_FILENAME

## Task Summary

| Property | Value |
|----------|-------|
| **Task Type** | email |
| **Priority** | normal |

## Task Content

[TASK CONTENT SUMMARY]

## Action Steps

1. [ ] Review task content
2. [ ] Determine required actions
3. [ ] Execute actions
4. [ ] Document results
5. [ ] Move to Done

## Approval Required

[Yes/No - explain]

---
*Auto-generated by AI Employee*
```

### Create Approval Request

For sensitive tasks, create an approval request file.

```markdown
# Create file in Pending_Approval/

---
type: approval_request
task_file: "TASK_FILENAME.md"
plan_file: "PLAN_FILENAME.md"
created: 2026-01-07T10:30:00Z
status: pending
---

# ⚠️ Approval Required

## Request Details

| Property | Value |
|----------|-------|
| **Task** | TASK_FILENAME |
| **Type** | email |

## Task Summary

[TASK SUMMARY]

## Instructions

**To Approve:** Move this file to `/Approved`
**To Reject:** Move this file to `/Rejected`

## Decision

*Add your decision here*
```

### Move Task to Done

After completing a task, move it to the Done folder.

```bash
# This is typically done by the user or orchestrator
# Move from Needs_Action to Done
# Move from Pending_Approval to Done (if approved)
```

### Update Dashboard

Update the Dashboard.md with current statistics.

```markdown
# Update Dashboard.md

1. Count files in each folder:
   - Needs_Action: X files
   - Plans: Y files
   - Pending_Approval: Z files
   - Done (today): W files

2. Update the stats table in Dashboard.md

3. Update "Last full refresh" timestamp
```

## Workflow

### Standard Task Processing

1. **Read** task from `Needs_Action/`
2. **Analyze** task type and priority
3. **Create** plan in `Plans/`
4. **Check** if approval needed:
   - If YES: Create file in `Pending_Approval/`
   - If NO: Execute task directly
5. **Execute** actions (after approval if needed)
6. **Move** task to `Done/`
7. **Update** Dashboard.md

### Approval Workflow

1. User finds file in `Pending_Approval/`
2. User reviews and adds decision
3. User moves to `Approved/` or `Rejected/`
4. AI processes approved tasks
5. AI moves completed to `Done/`

## Task Types

| Type | Description | Approval |
|------|-------------|----------|
| email | Email requiring response | Usually |
| file_drop | File dropped for processing | Sometimes |
| whatsapp | WhatsApp message | Usually |
| financial | Payment/invoice | Always |
| schedule | Meeting/request | Sometimes |

## Priority Levels

| Priority | Response Time | Auto-execute |
|----------|---------------|--------------|
| critical | Immediate | No |
| high | < 1 hour | No |
| normal | < 4 hours | Yes (if safe) |
| low | < 24 hours | Yes |
